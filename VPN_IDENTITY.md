# FlowWeave VPN 身份与轮换合同

本文件描述 VPN 模式的身份地基，不替代 [deploy/README.md](deploy/README.md) 的受控试点步骤。仓库现已有非特权 VPN 数据进程、事务化 TUN/路由/NAT helper 和 Type=notify client/server 单元，但 DNS、内部长期重连、身份运维工具、真实宿主安装验收与跨版本回退仍未完成。

## 信任链

VPN 会话必须连续通过以下检查，任意一步失败都不得启用 DATAGRAM 数据面：

1. TLS 1.3 服务端证书由客户端配置的服务端 CA 和 `server_name` 验证；
2. 服务端要求客户端证书，并由独立的客户端 CA 验证；
3. 双方协商 ALPN `flowweave-vpn/1`，禁止 0-RTT 应用数据；
4. 客户端叶证书 DER 的 SHA-256 指纹必须唯一属于一个已启用身份；
5. QUIC 必须真实协商 MPQUIC 和至少 1036 字节的 DATAGRAM，而不是只在 `HELLO` 中声明能力；
6. `FWC1` 控制流完成版本、能力、会话代际、虚拟地址和最大 IP 包长确认后，数据面才可启动；
7. 客户端数据句柄必须从受验证的 `ACCEPT` 和本地显式 ACL 构造，不得伪造证书指纹或服务端身份记录。

客户端 CA 只表示“证书签发来源可信”，不自动授予 VPN 权限。由该 CA 签发、但未登记指纹的证书会完成 TLS，然后收到稳定的 `unauthorized` 控制拒绝。

## 严格身份文件

示例见 `deploy/vpn-identities.json.example`。文件规则：

- UTF-8 JSON，`config_version` 当前只能为 `1`；
- 最大 1 MiB，必须是普通文件；
- Unix 上禁止 group 写/执行和任何 other 权限；单独运行数据层时可用专用用户 `0600`，接入 privileged 网络 helper 后应使用 `root:flowweave 0640`，让数据进程只读且不能反向修改下次 root prepare 的网络真值；
- 顶层、身份和 limits 对象中的未知字段、重复字段、缺失必需字段全部拒绝；
- 配置必须完整解析和通过全局冲突检查后才可原子替换当前注册表；失败保留旧注册表；
- 文件不得包含客户端或服务端私钥。

顶层字段：

- `server_ipv4` / `server_ipv6`：至少配置一个服务端虚拟地址；
- `identities`：最多 4096 个身份。

每个身份字段：

- `client_id`：1～64 字节，仅允许 ASCII 字母、数字、`-`、`_`、`.`；
- `fingerprints`：一到两个 64 位十六进制 SHA-256 指纹；两个槽用于证书轮换重叠期；
- `enabled`：`false` 时新控制握手返回 `identity_disabled`，活动会话协调器同时关闭该身份的当前代际；
- `client_ipv4` / `client_ipv6`：至少一个，并且对应服务端必须配置相同地址族；
- `allowed_destinations`：显式目标 CIDR 列表，空列表表示拒绝所有目标；CIDR 必须写成规范网络地址，例如 `10.0.0.0/8`，不能写 `10.0.0.1/8`；
- `limits`：显式连接数、包速率、字节速率和重组内存上限。VPN v1 的 `max_connections` 必须为 `1`，避免同一虚拟地址同时归属多个活动代际；重组内存不得小于一个最大 65,535 字节 IP 包。

全局拒绝条件包括重复 `client_id`、一个指纹属于多个身份、客户端虚拟地址重复、客户端地址等于服务端地址，以及身份声明了服务端未启用的地址族。

## 获取证书指纹

注册的是客户端叶证书的 DER 字节 SHA-256，而不是 PEM 文本、证书链或公钥指纹。可使用：

```bash
openssl x509 -in client-cert.pem -outform DER |
  openssl dgst -sha256 -r |
  awk '{print $1}'
```

写入配置前必须确认输出恰好为 64 位十六进制。默认日志和 `Debug` 输出不得打印完整指纹、证书、完整身份清单或私钥。

## 双证书轮换

轮换顺序固定为：

1. 用同一 `client_id` 同时登记旧、新两个指纹；
2. 完整验证候选配置，再原子替换注册表；
3. 客户端切换到新证书并完成真实 mTLS + `FWC1` 验证；
4. 从身份中删除旧指纹，再次验证和替换；
5. 之后使用旧证书发起的新会话必须收到 `unauthorized`。

增加第二指纹本身不关闭使用旧指纹的健康会话。删除当前会话使用的指纹、禁用/删除身份，或修改该身份的虚拟地址、ACL、资源 limits 时，协调器会先原子撤销其“当前代际”资格，再使用稳定应用关闭码关闭连接。新候选只有在真实 mTLS、`FWC1 ACCEPT` 写入成功且提交时仍符合最新身份文件后，才替换旧代际；任何握手、版本、身份或提交竞态失败都不能踢掉健康旧连接。

旧代际任务晚到的退出通知只能释放与其 generation 精确匹配的记录，不能误删后来已接管的会话。快速重连和双证书切换复用同一身份速率桶，不会恢复满额令牌。

应用关闭码固定为：

| 代码 | 原因 | 语义 |
| --- | --- | --- |
| `0x100` | `session_replaced` | 新代际已成功提交，旧代际退出 |
| `0x101` | `identity_revoked` | 身份禁用/删除、活动指纹撤销或指纹被重新归属 |
| `0x102` | `policy_changed` | 虚拟地址、ACL 或资源 limits 已变化，必须重新握手 |
| `0x103` | `session_commit_rejected` | `ACCEPT` 后提交时发现身份状态已变化；只关闭候选 |
| `0x104` | `server_shutdown` | 服务端受控关闭全部活动代际 |
| `0x105` | `control_failed` | 客户端控制流格式、读取或服务端响应验证失败 |
| `0x106` | `control_rejected` | 服务端已返回稳定 `REJECT`，双方结束该连接 |

## 资源与数据策略合同

- 包和字节 token bucket 在一个身份的上下行之间共享，初始 burst 为一秒额度；额度按外层 `FWI1` QUIC DATAGRAM 的次数和完整字节数在解析前消耗，畸形、重复或伪造输入不能免费占用 CPU；
- 每个身份拥有独立的速率锁和数据句柄，不同客户端逐包处理不再经过会话协调器的全局互斥锁；同一身份的新旧证书和快速重连继续共享 token bucket，不能靠换证书或代际清零；
- 所有身份只共享原子全局重组账本：默认最多 64 MiB、8192 个未完成包，硬上限为 1 GiB、65,536 个未完成包；单身份仍受自身 `max_reassembly_bytes` 和 1024 包上限约束；
- 数据句柄在分配 fragment 内存前取得内部临时 RAII 额度，真实 `VpnReassembler` 返回后按缓冲字节与未完成包的前后差精确结算；完成、完全重复、冲突、`total_len` 改变、淘汰、超时和 Drop 都会归还多余额度；
- 新代际提交时先使旧句柄 stale 并同步清空其重组状态，再激活新句柄；禁用、策略变化、正常释放和服务端关闭使用同一清理路径，Drop 只作为最终兜底；
- 上行完整 IP 包的源地址必须精确等于身份分配地址，目标必须命中显式 ACL；下行目标必须等于该身份地址，返回源地址必须命中同一 ACL；
- `ACCEPT.max_ip_packet_len` 是实际数据合同，不是只在握手中检查的装饰值：出站在分片和扣额前拒绝超限完整包；入站先按外层 DATAGRAM 扣额，再拒绝超限 `total_len`、清除同 packet-id 的旧分片并精确归还全局重组额度；
- 速率额度先于 IP 头、源地址和 ACL 检查消耗，已认证恶意客户端不能靠持续发送畸形或伪造包免费占用解析 CPU；
- 指标分别记录额度准入、实际策略放行、stale generation、速率拒绝、源地址伪造、ACL、下行错投和当前/峰值重组内存，不把“通过 token bucket”误写成“已经转发”。

高吞吐热路径和真实 QUIC DATAGRAM 运行器的关键边界已经完成：控制面全局锁只参与建连、替换、重载和关闭；数据面按身份分片；外层 `FWI1` 准入、严格解析、协商包长、双向独立 packet-id 空间、真实重组、原子全局账本和完整 IP 策略在同一个入口闭合。服务端句柄来自已授权身份，客户端句柄由一个可复用工厂从受验证的 `ACCEPT`、本地 ACL 和本地资源上限创建，两边复用同一数据核心；客户端候选代际与旧代际重叠时继续共享速率桶、全局重组预算和指标，不能靠重连翻倍额度。每个代际只能绑定一个运行器；NoQ 收发侧使用包数/字节双重有界队列，入站消费者停顿时丢新包并继续读取 QUIC，出站使用等待型 DATAGRAM 背压而不静默挤掉旧片。

## 当前证据和剩余边界

现有自动测试覆盖：

- 双指纹同时映射同一身份；增加轮换指纹保持现有会话，删除活动指纹会立即撤销当前代际；
- 未知、禁用、错误客户端 CA 三类拒绝路径；
- 重复身份、指纹、虚拟地址、非规范 CIDR和越界 limits；
- 严格 JSON、重复/未知字段、受保护文件权限和失败重载保留旧状态；
- 真实 loopback QUIC mTLS 上的 `HELLO/ACCEPT/REJECT`；三连接测试证明失败候选不扰动旧会话，成功候选收到 `ACCEPT` 后才替换；
- 实际 MPQUIC、DATAGRAM 下限、ALPN、虚拟地址和会话代际检查；
- 并发 token bucket 不超发、快速重连不重置、两个身份互不阻塞、全局重组字节/包数原子上限和内部 RAII 精确结算；
- 外层 DATAGRAM 先计费，随后覆盖乱序完成、完全重复、冲突、`total_len` 改变、单片完整包、身份内淘汰、超时和双向相同 packet-id；
- 代际替换、身份禁用、正常释放、服务端关闭和句柄 Drop 都会使旧句柄 stale 并把当前重组账本归零；
- 20,000 组确定性对抗 DATAGRAM 序列逐步核对“句柄实际缓冲 = 全局原子账本”，没有越界、超额或泄漏；
- 真实 loopback NoQ 连接上的双向 IPv4/IPv6 分片传输、相同代际重复绑定拒绝、无流量 stale 代际定时退出，以及入站队列满时不阻塞 QUIC 接收；
- 客户端工厂会验证 `ACCEPT`、地址族、本地 ACL 和全局资源上限，并直接绑定服务端分配地址、会话代际和协商包长；重复 ACL、缺失地址族和非法资源值全部失败关闭；两个同时存活的重连代际已验证共享同一包速率桶和全局未完成包预算；
- 一条真实组合链路已经串通 mTLS、`FWC1`、受管服务端会话、客户端工厂和双方 DATAGRAM 运行器，完成分片 IPv4 上行、IPv6 下行、协商包长超限拒绝；服务端关闭和成功代际替换都会使旧运行器退出；
- IPv4/IPv6 严格头检查之后的上行源地址防伪、双向 ACL 与下行错投拒绝；
- 身份文件失败重载保持活动会话，成功禁用重载原子撤销并关闭当前连接。

尚未完成身份配置 SIGHUP 接线、专用身份运维工具、DNS 接管与回退、内部长期重连、长期多客户端资源攻击/故障注入、真实宿主安装/升级门控，以及 nightly + AddressSanitizer 门控。现有产品命令、TUN/路由/NAT 事务和 systemd 生命周期只把身份地基接入受控试点，不能据此证明 VPN 已达到生产可部署标准。
