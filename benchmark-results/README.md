# 基准原始结果

这个目录保存实际运行得到的原始基准数据，不放手工编造的示例。

- `2026-07-11-scheduler-screening.csv`：删减候选前，四种多路径调度与两条单路基线，在平衡和异构线路下使用 5 对固定随机种子的 2 MiB 候选筛选。它证明 MinRTT 在异构线路只有 1/5 轮达到“快 15%”，因此已被删除。该筛选不等同于 `BENCHMARK.md` 规定的 20 秒或 64 MiB 最终验收。
- `2026-07-11-scheduler-screening-survivors.csv`：以后重跑筛选时保存 NoQ 默认、轮询和预计最早送达三个保留项，避免覆盖删减前的原始证据。

CSV 中每一行都是一次真实运行，包含种子、参赛者、吞吐量、两条线路首次发送的 STREAM 数据、重传数据、UDP 总线速和相对最佳单路的比例。失败轮次不得手工删除。
