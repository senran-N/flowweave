# 基准原始结果

这个目录保存实际运行得到的原始基准数据，不放手工编造的示例。

- `2026-07-11-scheduler-screening.csv`：删减候选前，四种多路径调度与两条单路基线，在平衡和异构线路下使用 5 对固定随机种子的 2 MiB 候选筛选。它证明 MinRTT 在异构线路只有 1/5 轮达到“快 15%”，因此已被删除。该筛选不等同于 `BENCHMARK.md` 规定的 20 秒或 64 MiB 最终验收。
- `2026-07-11-scheduler-long.csv`：两条单路、NoQ 默认、轮询和预计最早送达，在平衡与异构线路中各跑 5 对固定种子。每场使用 release 构建，单连接预热 2 秒后计时至少 20 秒，共 50 场真实传输。结果证明轮询只在平衡线路通过，异构线路仅 1/5 轮过线；预计最早送达在异构线路 0/5 过线，两者都必须删除。
- `2026-07-11-delivery-rate-weighted-screening.csv`：两条单路、NoQ 默认和“交付速率加权”候选，共 40 场真实 2 MiB 传输。该候选按 `拥塞窗口 / 平滑 RTT` 加权，并保留慢路探索流量。平衡线路 5/5 轮过线，中位为最佳单路的 171.6%；但异构 8+25 Mbit/s 线路 0/5 轮过线，中位仅 96.2%，最差 83.6%，UDP 线速中位却达到应用数据的 1.09 倍。它把两路错误分成接近 1:1，已经完整删除；由于没通过短筛，没有运行长测。
- `2026-07-11-ack-ecf-screening.csv`：两条单路、NoQ 默认和 ACK-ECF，共 40 场真实 2 MiB 传输。ACK-ECF 用应用数据 ACK 估计逐路径交付率，再计算预计完成时间。平衡线路 5/5 轮过线，相对最佳单路中位 161.2%；异构 8+25 Mbit/s 线路 0/5 轮过线，中位 95.5%，最差 74.7%，慢路仍承担中位 44.0% 的首次数据。结果证明普通 ACK 速率会学习调度器自己的投喂比例，而不是未饱和路径的容量上限；候选已完整删除，没有运行长测。
- `2026-07-12-pto-hedge-screening.csv`：同一 NoQ 版本的默认恢复与 FlowWeave PTO 跨路径对冲，使用 5 对固定种子分别完成黑洞恢复、正常网络误触发和高丢包抗误判，共 30 场真实传输。候选黑洞首字节恢复 5/5 低于 1 秒，P95 为 393.00 ms；正常网络吞吐和 UDP 相对基线中位分别为 99.45% 与 100.68%；高丢包 UDP 相对基线中位为 100.40%，5/5 至少保留一条路径，因此同时通过预先锁定的三组短筛。CSV 同时保留完整 256 KiB 校验完成时间，不能把冷备用路慢启动时间混成断流恢复时间。

CSV 中每一行都是一次真实运行，包含种子、参赛者、吞吐量、两条线路首次发送的 STREAM 数据、重传数据、UDP 总线速和相对最佳单路的比例。失败轮次不得手工删除。

长测还记录 CPU 和进程常驻内存。CPU 是计时窗口内的进程 CPU 时间；内存为同一测试进程内采样到的峰值 RSS。由于 Rust 分配器可能保留前一场释放的内存，后跑参赛者的 RSS 会继承进程历史，因此这一列可用于发现异常增长，但最终跨程序内存比较必须让每个参赛者运行在独立进程中。
